ScreenSense AI — 사용자 의도 감지 메커니즘

Computer Use 모델 + LLM + 특허 알고리즘 · DOM 스크립트 주입으로 앱 화면 직접 분석

핵심: 사용자가 클릭하기 전에 AI가 의도를 먼저 파악한다. 클릭·체류·커서 패턴으로 "막혔는지 / 탐색 중인지 / 완료 중인지" 실시간 분류.

3단계 작동 구조

단계 명칭 입력 데이터 수행 내용
1 Context
맥락 인식		 현재 화면 상태, UI 요소 위치, 앱 화면 ID 사용자가 어느 화면에서 무엇과 상호작용하는지 감지
2 Intent
의도 추론		 클릭 패턴, 체류 시간, 커서 이동, 입력 여부 막혀있는지 / 탐색 중인지 / 작업 완료 중인지 3가지로 분류
3 Action
실시간 반응		 Context + Intent 결합 결과 안내 팝업 / 도움말 / 자동화 / 외부 AI 중 적절한 것 즉시 트리거
기술 구현 방식: Whatfix Studio(Chrome 확장) → 대상 앱에 콘텐츠 스크립트 주입 → DOM 직접 접근 ID·Class·Attribute 값으로 UI 요소 식별 → LLM이 강건한 CSS 셀렉터 자동 생성 앱 UI가 위치·언어·색상 변경돼도 → 자동 재적응 (셀렉터만 갱신, 콘텐츠 수정 불필요) iframe, Shadow DOM 내 중첩 요소도 다층 알고리즘으로 추적

② Guidance Agent — 행동 기반 실시간 개입

사용자 행동 신호(클릭·체류·이탈) 실시간 감지 → 막힘 포인트에 자동 가이던스 삽입

핵심 원칙: 막히는 순간에 개입하는 것이 막히기 전에 가르치는 것보다 효과적이다. Guidance Agent는 브라우저 확장(Chrome Extension) 형태로 실제 앱 위에 오버레이되어 작동한다.

🔧 6단계 저작 워크플로우 (콘텐츠 제작자 기준)

1
Studio 확장 설치 및 활성화

Chrome Web Store에서 Whatfix Studio 확장 설치 → 대상 웹앱 접속 → Studio 아이콘 클릭하여 편집 모드 진입. 실제 운영 앱 위에서 직접 작업한다.

2
가이던스 유형 선택

5종 UI 중 상황에 맞는 형태 선택: Flow(단계별 안내) / SmartTip(단일 툴팁) / Pop-up(알림) / Self-help(검색 위젯) / Sidebar(패널형). 선택 즉시 편집 패널 활성화.

3
DOM 요소 클릭으로 앵커 지정

가이던스를 표시할 UI 요소(버튼·필드·메뉴 등)를 화면에서 직접 클릭 → LLM이 CSS 셀렉터 자동 생성 및 XPath 저장. 코드 작성 불필요.

4
메시지 작성 및 미디어 추가

텍스트·이미지·GIF·영상 삽입. AI Auto-Create 기능으로 스크린샷 기반 설명 자동 생성 가능. 다국어 번역(45개 언어)도 원클릭으로 처리.

5
트리거 조건 설정

자동(페이지 로드) / URL 매칭 / 클릭·호버 이벤트 / 수동(사용자 요청) / 역할 기반(특정 사용자 그룹) 중 선택. 복합 조건도 AND/OR 로직으로 구성 가능.

6
게시 및 A/B 테스트

Preview 확인 후 Publish → 실시간 적용(앱 재배포 불필요). 복수 버전 A/B 테스트 설정 → Insights Agent가 클릭률·완료율 자동 측정.

① Parsing Engine — 모든 형식 콘텐츠 읽기

SCORM·PDF·MP4·MP3·PPT·Word·HTML5·ePub 전 형식 구조화 · Storyline·Rise·Captivate 지원

🔧 5단계 콘텐츠 처리 파이프라인

1
업로드 / 연동

파일 직접 업로드 또는 클라우드 스토리지(Google Drive·SharePoint·AWS S3·Dropbox) 연동. SCORM 패키지(.zip) 업로드 시 압축 자동 해제.

2
형식별 파싱

SCORM — imsmanifest.xml 먼저 파싱: <organizations> 태그로 콘텐츠 계층(단원·차시) 추출 → <resources> 태그로 실제 콘텐츠 파일(HTML·SWF·JS) 식별 → 각 리소스 순차 읽기. PDF — OCR + 레이아웃 분석으로 섹션 구분. MP4/MP3 — Whisper 기반 자동 자막(STT) 변환 후 구조화. PPT — 슬라이드별 텍스트·이미지·노트 분리 추출.

3
LLM 분석

파싱된 텍스트 → LLM 전송. 학습 목표·핵심 개념·전제 지식·난이도 자동 추출. 콘텐츠를 의미 단위(청크) 분할 — 평균 청크 300~500토큰 기준.

4
벡터 DB 저장

각 청크를 임베딩 벡터로 변환 → Vector DB(Pinecone 계열) 저장. 메타데이터 함께 저장: 콘텐츠 ID·형식·파싱 일시·스킬 태그 목록.

5
결과물 생성

① Simulate — AutoBuild AI 시뮬레이션 자동 생성

from ETU · 텍스트 프롬프트 → 분기형 시나리오 자동 완성 · 수개월 → 수 분

핵심 원칙: 시나리오 개발의 병목은 사람이다. AI가 초안을 만들면 전문가는 검수와 고도화에만 집중할 수 있다.

🔧 4단계 AutoBuild 생성 흐름

1
정의 (Define)

학습 목표·대상 직군·역할(AI 페르소나)·시나리오 난이도 설정. 구조화 폼에 입력하거나 자연어로 기술 가능. 예: "팀장 역할의 학습자가 실적 부진 팀원과 코칭 면담하는 상황, 중급 난이도"

2
AI 생성 (Generate)

입력 기반으로 대화 트리 자동 생성: 오프닝 → 분기 노드(최대 3~5개 선택지) → 각 선택지별 AI 반응 → 분기 결과(긍정/중립/부정). 동시에 루브릭(평가 기준) 초안 자동 생성.

3
검토·수정 (Review)

시각적 대화 트리 편집 UI에서 HRD 전문가가 대화 흐름 검수. 노드 추가·삭제·텍스트 수정·분기 조건 조정. 실제 대화 미리보기(Preview) 기능으로 학습자 경험 사전 확인.

4
게시 (Publish SCORM)

완료 시 SCORM 1.2/2004 패키지로 내보내기 → 기존 LMS에 바로 업로드 가능. 또는 Skillwell 플랫폼에서 직접 운영. Skill Tags 자동 부착 후 Adapt 엔진과 연동.

② Adapt — 적응형 AI 학습 경로 (Realizeit 엔진)

from Realizeit · Determine Knowledge™ — 이미 아는 것 건너뜀 · 학습 시간 40% 절약 · Skills Mastery 20~50% 향상

핵심 원칙: 모든 학습자에게 같은 경로를 강요하지 않는다. Determine Knowledge™로 이미 아는 것은 건너뛰고, 부족한 것은 깊이 판다.

🔧 Determine Knowledge™ 3단계 작동 원리

1
적응형 사전평가 (Adaptive Pre-Assessment)

정해진 순서 없이 응답에 따라 다음 문항이 바뀌는 CAT(Computer Adaptive Testing) 방식. 최소 문항(보통 5~15개)으로 학습자의 현재 지식 상태 추정. 같은 30문항 퀴즈를 모두에게 주는 방식 대비 시간 50% 절약.

2
잠재 지식 상태 추정 (Knowledge State Estimation)

Bayesian Knowledge Tracing 알고리즘 기반. 각 스킬/개념에 대한 숙련도를 0~100% 확률로 추정. 입력 데이터: 사전평가 응답 + 이전 학습 이력 + Simulate 시나리오 선택지 기록 + Skill Tags.

3
개인화 경로 배치 (Path Assignment)

추정 숙련도 기반으로 커리큘럼 그래프 상 최적 시작 지점 결정. 숙련도 80% 이상 개념 → 스킵. 50~79% → 압축 복습. 50% 미만 → 전체 학습. 학습 진행 중에도 실시간 재계산 → 경로 즉시 재조정.

🏷️ Skill Tags 작동 방식

Skill Tags는 시나리오의 각 선택지에 역량 레이블을 미리 부착해두는 방식이다. 학습자가 선택지를 클릭할 때마다 해당 Tag가 기록되고, 이를 집계해 역량 달성 여부를 판단한다.

1
Tag 부착 (설계 단계)

AutoBuild에서 각 분기 선택지에 Skill Tag 레이블 부착. 예: 선택지 A → #공감적경청, 선택지 B → #목표설정, 선택지 C → #저항처리. 한 선택지에 복수 태그 가능.

2
Tag 수집 (실습 단계)

학습자가 시나리오 진행 시 선택지별 Tag 자동 누적. 전체 시나리오 완료 시 학습자별 Tag 선택 패턴(어떤 역량을 얼마나 발휘했는지) 데이터화.

3
집계 → 3레벨 리포트 생성

개인 / 팀 / 조직 3레벨로 집계. 역량별 달성률·패턴·개선 필요 영역 자동 시각화.

A

학습 흐름 안에 녹이기

from Whatfix · Guidance Agent 원칙

교육을 별도 시간·공간에서 하지 않고, 학습자가 실제 과제를 수행하는 흐름 안에 삽입한다.

  • 1 시뮬레이션 중 실시간 코칭 — 학습자가 롤플레잉 중 막히면 AI가 즉시 힌트 제공. 세션 종료 후 피드백이 아니라 막히는 그 순간에 개입.
  • 2 온보딩 중 즉시 안내 — 플랫폼 첫 접속 시 "이 버튼은 이렇게 씁니다" 형태의 인터랙티브 가이드.
  • 3 학습자 행동 데이터 수집 — "어느 단계에서 얼마나 오래 머무는가"로 콘텐츠 품질 지속 개선.
B

이미 있는 것에서 가치 꺼내기

from MetaLark.AI · 자산 역량 매핑 원칙

새 콘텐츠를 만들기 전에, 기존 자산을 AI로 분석해서 어디에 공백이 있는지 먼저 파악한다.

  • 1 기존 교육 자산 역량 지도 작성 — 고객사 보유 콘텐츠를 분석해서 "현재 커버된 역량 지도"를 만들고, 공백을 채우는 솔루션을 제안.
  • 2 재활용 콘텐츠 파이프라인 — 기존 강의 영상·PPT·케이스 스터디를 AI로 재구조화해서 새 과정에 모듈로 편입.
  • 3 제안서 현황 진단 서비스화 — "귀사 교육 자산 중 역량 커버율이 X%입니다. 이 역량들이 공백입니다" → 솔루션 필요성 데이터로 제시.

D

시나리오 기획 대신 시나리오 생성

from Skillwell · AutoBuild + Verified Capability 원칙

수개월 걸리던 시뮬레이션 제작을 AI가 수 분 안에 초안으로 만든다. 사람은 검수와 고도화에만 집중. 그리고 완료가 아닌 역량 달성을 증명한다.

  • 1 AI 시나리오 초안 생성 — 역할·상황·분기 조건 입력 → AutoBuild가 대화 트리 초안 완성. HRD 전문가는 검수·조정에만 집중해 제작 기간 대폭 단축.
  • 2 개인화 학습 경로 자동 설계 — Determine Knowledge™로 학습자가 이미 아는 내용은 건너뜀. 부족한 역량에만 집중하게 해 학습 시간 40% 절약.
  • 3 역량 달성 증명 — 수료(completion) 대신 Verified Capability. "교육을 받았다"가 아닌 "역량을 갖췄다"를 데이터로 증명 → 고객사 성과 가시화.

시나리오 A — AI 롤플레잉·코칭 솔루션 고도화 (3개 플랫폼 통합)

현재: 사전 교육 → 시뮬레이션 → 종료 후 피드백 (단절) 벤치마킹 적용 후: ① [기획] MetaLark 방식: 기존 시나리오 라이브러리 분석 → 역량 공백 자동 발굴 → 만들어야 할 것만 특정 ② [제작] Skillwell AutoBuild 방식: 역량 공백 기반 시나리오 AI 자동 생성 → 전문가 검수만 ③ [실습] Whatfix Guidance Agent 방식: 시뮬레이션 중 막힘 감지 → 즉시 힌트 삽입 (ScreenSense AI 개념) ④ [검증] Skillwell Verified Capability 방식: 역량 달성 데이터 기반 고객사 성과 보고
📋

시나리오 B — 고객사 제안서 현황 진단 차별화

기존: 고객 인터뷰 + 설문 → 현황 진단 (주관적) 벤치마킹 적용 후: ① MetaLark 방식: 고객사 보유 교육 콘텐츠 분석 → 역량 커버율 데이터 기반 진단 ② 수치 제시: "현재 리더십 역량 콘텐츠 40%, 디지털 역량 콘텐츠 5%" ③ 연결: 공백 역량 → 우리 솔루션으로 채우는 제안으로 연결
⚙️

시나리오 C — 콘텐츠 제작 효율화

기존: 매 과정마다 처음부터 콘텐츠 제작 벤치마킹 적용 후: ① 전수 분류: 기존 강의 자료·케이스 스터디 전수 분류 (MetaLark 방식) ② 재사용 모듈 발굴: 재사용 가능 모듈 발굴 → 신규 과정에 편입 ③ 공백만 신규 제작: Skillwell AutoBuild 방식으로 시나리오 AI 초안 생성 → 전문가 검수만
🚀

시나리오 D — 3개 플랫폼 개념 통합: L&D 전 주기 혁신

STEP 1 — 진단 (MetaLark.AI): 고객사 교육 자산 전수 분석 → 역량 커버 지도 + $9B Content Chaos 맥락 → 무엇이 공백인지 데이터화 STEP 2 — 제작 (Skillwell AutoBuild): 공백 역량 기반 AI 시뮬레이션 초안 자동 생성 → HRD 전문가 검수·조정 → 수개월 → 수 분 STEP 3 — 개인화 (Skillwell Adapt + Determine Knowledge™): 학습자별 사전 역량 감지 → 이미 아는 것 스킵 → 부족한 것 집중 → 학습 시간 40% 절약 STEP 4 — 현장 개입 (Whatfix ScreenSense + Guidance): 실습 중 막힘 실시간 감지 → 즉시 힌트 삽입 → 이탈 방지 STEP 5 — 검증 (Skillwell Verified Capability): 역량 달성 데이터 기반 고객사 성과 보고 → "수료율"이 아닌 "역량 달성률"로 재계약 근거
